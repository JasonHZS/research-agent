## 任务

请将上述研究发现整理为该章节的内容，包含:

1. **章节内容**: 结构化的 Markdown 格式内容，可直接用于最终报告
2. **信息来源**: 论文标题、URL 等（确保可追溯）
3. **关键发现**: 3-7 条最重要的发现摘要

## 压缩原则

1. **保留实质**: 保留具体的技术细节、数据、方法论
2. **去除冗余**: 移除重复信息、无关内容、营销性描述
3. **保持可追溯**: 每个关键论点都要标注来源
4. **结构清晰**: 使用 Markdown 格式，包含子标题和要点

## 输出格式

请严格按照以下 JSON 格式输出:

```json
{
  "title": "章节标题（与输入保持一致）",
  "content": "## 概述\n\n该章节的主要内容...\n\n### 子标题1\n\n详细内容...\n\n### 子标题2\n\n更多内容...",
  "sources": [
    "[论文标题](ArXiv URL)",
    "[博客标题](URL)"
  ],
  "key_findings": [
    "发现1: 简洁描述",
    "发现2: 简洁描述",
    "发现3: 简洁描述"
  ]
}
```

## 注意事项

- `content` 应该是可读的连贯 Markdown 文本，可直接用于报告
- `content` 中的关键论点应标注来源
- `sources` 应该是完整的 URL 或可识别的引用
- `key_findings` 应该是具体的、可操作的信息摘要
- 如果原始发现中有矛盾信息，在 content 中说明不同观点
