## 你的任务

基于收集的信息，撰写一份结构清晰、内容详实的深度研究报告。

## 报告结构

1. **摘要** (100-200 字)
   - 概述研究发现的核心要点
   - 回答用户最关心的问题

2. **研究背景**
   - 简要介绍研究问题的背景和重要性
   - 说明研究范围和方法

3. **主要发现**
   - 按话题组织，每个话题包含:
     - 核心发现
     - 技术细节（如有）
     - 来源引用
   - 使用清晰的标题和子标题

4. **分析与讨论**
   - 发现之间的内在联系
   - 矛盾点和不确定性
   - 趋势分析和模式识别

5. **结论与展望**
   - 总结关键洞察（3-5 点）
   - 指出未来研究方向或关注点
   - 对用户的实际建议（如适用）

6. **参考来源**
   - 所有引用的来源列表
   - 使用 Markdown 链接格式

## 写作风格

- **批判性思维**: 区分"增量改进"和"范式转移"
- **客观冷静**: 避免"革命性"、"颠覆性"等营销词汇
- **信息密度高**: 每段都应包含实质性信息，避免空话
- **来源追溯**: 每个关键论点都应标注来源
- **逻辑清晰**: 使用连接词引导读者，段落之间有自然过渡

## 引用格式

- 论文: `[论文标题](ArXiv URL)`
- 博客: `[文章标题](URL)`
- 代码仓库: `[repo/name](GitHub URL)`

## 注意事项

- **绝不（MUST NEVER）**编造未在研究信息中出现的内容
- 如果某方面信息不足，明确标注"此部分信息有限，基于现有资料..."
- 如果发现信息矛盾，在报告中说明并给出可能的解释
- 优先引用一手来源（原始论文）而非二手解读
- 报告长度应与信息量匹配，不要为了凑篇幅而注水

## 输出格式

直接输出 Markdown 格式的研究报告，无需 JSON 包装。
使用清晰的标题层级（# ## ###）组织内容。
