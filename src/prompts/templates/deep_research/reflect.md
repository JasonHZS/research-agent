## 你的任务

评估当前收集的信息是否足以撰写一份高质量的研究报告。

## 评估维度

1. **覆盖度**：大纲中的每个章节是否都有足够的信息支撑？
2. **深度**：是否有足够的技术细节（而非仅有概述）？
3. **多样性**：信息来源是否多样（论文、博客、代码仓库）？
4. **一致性**：不同来源的信息是否有矛盾需要验证？
5. **时效性**：是否包含最新的进展（尤其是最近 6 个月内的）？

## 输出格式

请严格按照以下 JSON 格式输出：

```json
{
  "is_sufficient": true/false,
  "overall_score": "1-10 的评分，10 表示完全充足",
  "section_coverage": {
    "章节1标题": {

  },

```

## 决策逻辑

- `overall_score >= 7` 且无关键 gaps → `is_sufficient: true`
- 已达到最大迭代轮数 → `is_sufficient: true`（强制结束）
- 否则 → `is_sufficient: false`，继续检索

## 注意事项

- `next_queries` 应该是具体的、可执行的搜索词（如 `ti:RAFT` 而非 "找更多关于 RAFT 的信息"）
- 如果某个来源的信息有疑问，建议搜索原始论文验证
- 最多生成 3 个 `next_queries`，优先填补最关键的 gaps
