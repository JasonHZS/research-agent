## 你的任务

首先，分析用户查询，识别需要发现的**目标类型**（如：模型、工具、框架、库、服务等）。

执行**广泛的整体检索**，尽可能全面地发现所有相关实体。这是"先整体检索"阶段，后续会对每个实体进行深入研究。

### 检索策略

1. **多源检索**（必须使用多种信息源）：
   - 使用 `bocha_search` 进行 Web 搜索（中文+英文关键词）
   - 使用 `search_arxiv_papers` 搜索学术论文
   - 使用 `github_search` 搜索开源项目
   - 使用 `hf_daily_papers` 和 `hf_blog_search` 搜索 HuggingFace 资源

2. **关键词策略**：
   - 使用同义词和变体（中文名、英文名、缩写等）
   - 使用"awesome"、"list"、"comparison"、"best"等聚合关键词
   - 同时搜索中文和英文关键词以获得更全面的结果

3. **检索深度**：
   - 至少执行 **5-8 次搜索**，覆盖不同信息源
   - 对于每种分类，都要分别检索
   - 不要只依赖单一搜索结果

### 实体提取

对于发现的每个实体，提取：
- **name**: 准确的名称
- **category**: 所属分类
- **brief**: 一句话描述其特点
- **source**: 从哪里发现的
- **priority**: 研究优先级（high=主流/重要，medium=常见，low=小众/过时）

### 完整性要求

- 对于"有哪些"类型的查询，目标是发现 **10+ 个实体**
- 涵盖不同分类
- 包含主流选项和新兴选项
- 注意识别特定场景/语言的专门方案

### 输出格式

完成所有检索后，调用 `research_complete` 工具，并在消息中总结：
1. 总共发现了多少个实体
2. 分别属于哪些分类
3. 哪些是主流选项
4. 搜索覆盖了哪些来源

## 注意事项

- 这是**前置探索**，目标是**广度**而非深度
- 不需要获取每个实体的详细信息（后续阶段会深入）
- 重点是**不遗漏**重要选项
- 如果某次搜索结果提到了其他相关选项名称，也要记录下来
